Flow

Flow describes how an abstract human intention becomes a finalized, verifiable execution without requiring the user to understand infrastructure, routing, or counterparties.

High-Level Overview

Whytegod separates financial action into four distinct phases:

1. Intent Declaration
The user states what outcome they want — not how to achieve it.
2. Validation
The intent is checked for correctness, feasibility, and constraints.
3. Resolution
The system determines possible execution paths.
4. Execution
One valid path is executed and settled.

1. Intent Declaration

An intent is a structured statement of desire.

It may include:

• desired outcome
• acceptable constraints
• time boundaries
• trust assumptions

The user does not select routes, protocols, liquidity sources, or counterparties.

2. Validation

Before any execution is attempted, the intent is validated.

Validation ensures:

• the intent is internally consistent
• constraints do not conflict
• required conditions can be met

Invalid intents fail early and safely.

3. Resolution

Resolution is where intelligence replaces manual decision-making.

The system evaluates multiple possible execution paths, comparing them against the declared intent.

This phase is adversarial by design. Paths are assumed to be unreliable until proven otherwise.

4. Execution

Execution occurs only after a valid path is selected.

Execution engines are replaceable, auditable, and non-authoritative.

Failure does not compromise the system. It simply returns control to the resolution phase.

Finality

Once execution succeeds, the outcome is verifiable.

No hidden steps. No discretionary overrides. No silent failures.

What was declared is what occurred — or nothing occurs at all.
