Security Model

Whytegod minimizes trust by design.

  • No custody of user funds
  • No privileged execution roles
  • No hidden control paths

Threat Surface

The protocol assumes adversarial executors. Security emerges from verification, not trust.

Non-Goals

  • User protection from bad intent
  • Guaranteeing profitability
  • Preventing all failure

Failure is allowed.
Silent manipulation is not.
