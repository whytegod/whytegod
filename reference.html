Reference

This page exists for clarity, not persuasion.

Core Definitions

Protocol

A deterministic set of rules that governs behavior without requiring trust, discretion, or centralized control.

Intent

A user-expressed objective that describes *what* should happen, without specifying *how* it is executed.

Non-Custodial

A system that does not hold user assets, keys, or persistent state.

Deterministic

Given the same input, the protocol always produces the same output.

Fork

An independent continuation of the protocol when consensus cannot be reached.

Design Principles

  • Minimal surface area
  • No hidden state
  • No privileged roles
  • Explicit failure over silent behavior
  • Exit over control

What Whytegod Is Not

  • Not an exchange
  • Not a trading platform
  • Not a custody service
  • Not a financial intermediary
  • Not an investment product

Related Systems

The Whytegod Protocol is philosophically aligned with:

  • Bitcoin — rule-based monetary systems
  • Monero — privacy as default
  • Tor — minimization of trust
  • Unix philosophy — simple tools, clear behavior
