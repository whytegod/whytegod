Protocol

A protocol is a shared set of rules that anyone can follow without asking permission.

Whytegod is a protocol — not a company, not a broker, not a bank.

Intent

An intent is a statement of desired outcome, expressed in human language.

It describes what the user wants — not how the system should do it.

Execution

Execution is the process of fulfilling an intent.

In Whytegod, execution is handled by the system, not manually by the user.

Value

Value refers to any unit of worth recognized by the protocol.

It is not limited to a single coin or asset.

Decentralization

Decentralization means no single party can control outcomes.

Users retain the right to exit, fork, or verify independently.

Security

Security is the absence of surprise.

A secure system behaves predictably, even under stress.

Governance

Governance defines how changes are proposed, reviewed, and accepted.

In Whytegod, governance is intentionally limited.

User

A user is any individual interacting with the protocol.

No identity, status, or permission is required.

Fork

A fork is the right to copy the system and continue independently.

This is the final safeguard against abuse.

Whytegod

Whytegod is an idea expressed as software.

It exists to reduce complexity and restore human control.

If a term is unclear, the system is wrong — not the user.
